# Contribuire a GekOffice

## Ambito

GekOffice accetta contributi relativi a branding, packaging, documentazione, modelli, estensioni e patch mantenute dal progetto.

## Come contribuire

1. Apri una issue per descrivere problema, proposta o miglioramento.
2. Mantieni le modifiche piccole e facili da revisionare.
3. Invia una pull request con una descrizione chiara di cosa cambia e perche.
4. Collega eventuali issue correlate.

## Linee guida

- preferire pull request piccole
- documentare i cambiamenti di comportamento
- aggiornare test e documentazione quando necessario
- non mescolare modifiche non correlate nella stessa pull request
- rispettare le regole del progetto su marchi, branding e licenze

## Stile dei commit

Usare messaggi brevi e descrittivi. Esempio:

`docs: aggiunge roadmap iniziale`
