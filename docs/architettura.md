# Architettura del progetto

## Obiettivo

GekOffice e organizzato per separare chiaramente personalizzazioni, documentazione, modelli, estensioni e patch rispetto al codice upstream di LibreOffice.

## Strategia tecnica

Il progetto privilegia inizialmente:
- minimo numero di patch sul core
- massimo uso di configurazioni, modelli ed estensioni
- allineamento regolare con LibreOffice upstream
